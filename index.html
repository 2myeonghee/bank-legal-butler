가. 아파트집사 플랫폼의 현재 구조

아파트집사는 공동주택 입주예정자협의회(입예협)를 대상으로 질의 등록 → 인덱스 매칭 → 변호사 검토 → 답변 게시 → 레퍼런스 축적의 흐름으로 운영되는 AI+변호사 하이브리드 법률 자문 플랫폼이다. 집단등기, 하자, 건축 법률 등 공동주택 도메인에 특화된 RAG 인덱스와 반복 질문 자동 처리 구조가 핵심 자산이다.

핵심 발견: 플랫폼의 기술 구조(질의 구조화 → 인덱스 라우팅 → 변호사 할당 → SLA 관리 → 레퍼런스 축적)는 도메인에 무관하게 이식 가능하다. 현재 등기팀이 저축은행 등에 제공하는 수작업 자문을 이 구조로 대체하면 공수는 줄이면서 단가는 유지할 수 있다.

나. B2B 금융권으로 확장하는 이유

공급 측 근거 (로펌 입장)
  • 현재 등기팀 → 저축은행 자문이 수작업·구두 형태 → 레퍼런스 미축적, 담당자 교체 시 노하우 소멸
  • 변호사 1인이 반복 질문에 매번 새로 조사 → 시간당 원가 대비 수익성 낮음
  • 정액 구독 수익이 없어 매출 변동성 높음 → 월정액 구조로 안정 수익 확보 가능
  • 아파트집사 인프라 개발비 이미 투입 → 추가 개발 최소화로 ROI 극대화
수요 측 근거 (금융기관 입장)
  • 새마을금고 1,200개+, 저축은행 79개사 — 대부분 전담 법무팀 없음
  • 법률 이슈 발생 시 단건 의뢰 → 비용 예측 불가, 대응 지연
  • 반복적 질문(등기, 연체, 담보 처리)에 매번 착수금 지불 → 비효율
  • 2025년 이후 금융소비자보호법 강화 → 상시 법률 대응 필요성 급증
  • 지점 담당자 수준에서 처리 가능한 법률 판단 기준이 없어 본점 에스컬레이션 과다

인천 남동공단 등 중소기업 — 계약, 노무, 규제 질의 다양. 1·2차 레퍼런스 충분히 축적된 뒤 진입. 질의 다양성과 원가 회수 리스크를 고려해 단계 진입.

다. L2 — 변호사 검토 상세

사실관계에 따라 답변이 달라지는 질문. 계좌번호·담보물·채무자 정보 등 구체적 맥락을 받아야 정확한 답변이 가능. 담당군 변호사가 48시간 내 답변.

나. 실행 로드맵

PHASE 1
파일럿 구축 및 첫 계약
0 ~ 3개월

기술

  • RAG 데이터 교체 (금융법 6종 파싱·임베딩)
  • 질문 입력 폼 재설계
  • 담당군 라우팅 4개 그룹 설정
  • L1 인덱스 30~50건 초기 구축

영업·운영

  • 기존 등기팀 거래 저축은행 1~2개사 파일럿 제안
  • 3개월 무료 파일럿 → 유료 전환 조건 명시
  • 모든 질의 L2 처리 → 레퍼런스 50건 이상 축적
  • 파일럿 완료 후 레퍼런스·성과 정리 → 제안서 자료화
PHASE 2
L1 자동화 가동 및 고객 확장
3 ~ 9개월

기술

  • L1 자동화 가동 (목표: 전체 질의의 40% 처리)
  • 화이트라벨 고객 포털 구축 (기관별 로고·색상)
  • 월 정기 법률 리포트 자동 생성 기능
  • 자동 규제 변경 알림 (금융위·금감원 공고 RSS)

영업·운영

  • 저축은행·새마을금고 5~10개사 STANDARD 계약 목표
  • 파일럿 고객 추천 프로그램 운영
  • 월 구독 수익 500만원 이상 달성
  • 2차 타깃(상가연합회·프랜차이즈) 파일럿 1개 착수
PHASE 3
플랫폼 안정화 및 타깃 확장
9 ~ 24개월

기술

  • L1 자동화율 60% 이상 달성
  • ENTERPRISE 전담 변호사 배정 시스템
  • 2차 도메인(상가·프랜차이즈) RAG 추가
  • API 연동 — 고객사 내부 시스템 연결

영업·운영

  • 고객 20개사 이상 / 월 수익 2,000만원 목표
  • 새마을금고 중앙회 연계 — 지점 일괄 계약 추진
  • 3차(중소기업) 시장 진입 검토
  • 플랫폼 외부 판매(타 로펌에 라이선스) 검토
